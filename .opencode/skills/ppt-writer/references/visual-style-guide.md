# PPT Writer Visual Style Guide

## 总体原则

1. 内容先于装饰。视觉元素必须帮助用户理解结构、关系、流程或重点。
2. 一页一个任务。页面角色不清楚时，先重写页面，不要直接排版。
3. 标题表达判断。避免全篇“背景”“问题”“方案”式标题。
4. 控制文本密度。PPT不是Word，正文应可被听众在5-8秒内扫读。

## 常用版式

- 封面：大标题 + 副标题 + 一条视觉线索。
- 章节页：章节编号 + 章节标题 + 本章核心问题。
- 两栏页：左问题右方案、左现状右目标、左风险右对策。
- 卡片页：3-4个并列要点，每个卡片包含短标题和1-2句解释。
- 流程页：3-5步，不超过6步。
- 矩阵页：用于课程设计、能力映射、威胁模型、责任分工。
- 结论页：3条以内结论 + 下一步。

## 技术/课程类PPT的建议

- 概念先定义，再给例子，再推理分析，再进入动手或方案。
- 对复杂架构，先给总图，再逐层展开，不要一次塞满所有细节。
- 对安全课程，显式区分资产、威胁、攻击路径、防护控制、验证方式。
- 对工程方案，显式区分控制平面、执行平面、数据/状态存储、审计反馈。

## 中文排版

- 不要在中文与英文/数字之间强制加空格，除非品牌或术语规范要求。
- 使用中文全角标点。
- 页面内术语保持一致，例如不要在同一deck内混用“智能体/Agent/代理”。
- 避免长括号解释；必要定义放到备注或附录。

## 视觉QA问题

- 标题换行导致装饰线或图形错位。
- 正文过密，卡片内行距不足。
- 图标和文字低对比度。
- 页脚、来源说明和正文重叠。
- 章节页风格与内容页割裂。
- 同类页面卡片大小、间距、对齐不一致。
