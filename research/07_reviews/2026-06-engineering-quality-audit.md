## Risks

1. **README reproduction command fails as written.**
   - Path: `D:\MyWorkSpaces\contract-driven-harness-study\README.md` lines 35-53.
   - README says `cd research/04_methods/scripts` then run. Manifest output paths are relative to the repo root (e.g., `research\05_analysis\...`), so running from `scripts/` fails with file-not-found. Scripts also import siblings by bare module name, so `PYTHONPATH` is required.
   - Actual runnable form: from repo root, set `PYTHONPATH=research\04_methods\scripts`.
   - Reviewer impact: weakens first impression of the reproducibility package.

2. **Statistical thresholds and decision constants are heavily hard-coded.**
   - `analyze_stage_b_v53_explicit_transition_delta.py` lines 173, 177, 192, 198: `0.20` engineering-effect threshold, `<0.05` strong-support threshold.
   - `analyze_stage_b_v54_delta_stability.py` lines 50, 59, 69-71, 90-94: `38/40`, `7/8`, `39/40`, exact McNemar `0.500`.
   - All are documented in preregistration, but there is no machine-readable experiment spec; modifying a threshold requires editing multiple scripts. Post-hoc drift cannot be detected by tooling.

3. **Fixtures and evaluator are co-generated, creating fixture-fit / evaluator-overfit risk.**
   - `build_stage_b_v53_explicit_transition_delta.py` and `evaluate_stage_b_v53_explicit_transition_delta.py` share semantics: known-bad outputs are generated by the build script using the same golden output and expectation rules; the evaluator's forbidden-model-surface strings also come from the build script.
   - Paper V4 Section 5.5 already concedes "known-bad suite covers only the failure modes that were anticipated or observed". This is a methodological threat, not an implementation bug.

## Overall Engineering Grade

**B.**

Project has runnable tests, complete Stage B v5.4 artifact chain, faithful evidence ledger, and a reproduction that matches the frozen record. But README reproduction command is broken, 28 source_ids are unindexed, key thresholds are hard-coded, no dependency declaration, and `02_notes/`/`qa/` remain templates. These are fixable reproducibility hazards that prevent presenting the package as a flawless "A-grade reproducibility package" in auxiliary review material.
