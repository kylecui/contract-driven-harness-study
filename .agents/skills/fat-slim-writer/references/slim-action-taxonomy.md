# Slim Action Taxonomy

Slim is not only deletion. Use five editing actions, chosen based on what the content needs.

## DELETE

Use when:
- Content does not support the argument
- Content repeats other content already covered elsewhere
- Content weakens the structure or pacing
- Content is filler that adds no information

## COMPRESS

Use when:
- Content is useful but too verbose
- Same point can be made in fewer words without losing meaning
- Multiple paragraphs can be condensed into one
- Lists can be shortened while preserving key items

## MERGE

Use when:
- Two or more sections express similar points
- Combining them creates one stronger, more focused section
- Cross-chapter redundancy is found during cross-chapter audit
- Related arguments scattered across sections should be unified

## MOVE

Use when:
- Content is valuable but appears in the wrong chapter or section
- Supporting material belongs in an appendix rather than main text
- Background information interrupts the main flow
- Content is better as a footnote, sidebar, or supplementary section

## REWRITE

Use when:
- The idea is valuable but the expression is unclear
- Language is too generic, too casual, or inconsistent with writing contract
- Jargon level does not match the audience
- Tone or perspective needs adjustment without changing the substance

## Decision Priority

When multiple actions could apply, prefer in this order:
1. REWRITE (preserve content, fix expression)
2. COMPRESS (preserve content, reduce length)
3. MOVE (preserve content, relocate)
4. MERGE (combine with other content)
5. DELETE (remove entirely)
