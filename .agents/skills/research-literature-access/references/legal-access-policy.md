# Legal Access Policy

## Core Principle

All literature access must be legal. This skill helps find the best legal path to full text, not circumvent access controls.

## Free-First Priority Order

| Priority | Source Type | Examples |
|---|---|---|
| 1 | User-uploaded file | PDF already in project directory |
| 2 | Official open version | Gold/Diamond OA, government docs |
| 3 | Preprint | arXiv, bioRxiv, SSRN, TechRxiv |
| 4 | Author homepage | Personal/lab website self-archive |
| 5 | Institutional repository | University digital library |
| 6 | Open database full text | PubMed Central, CORE, Unpaywall |
| 7 | Institutional subscription | Library, company, association |
| 8 | Personal purchase | Publisher account, bookstore |

## Forbidden Actions

- Accessing pirated copies (Sci-Hub, LibGen for copyrighted works, etc.)
- Circumventing DRM or access controls
- Using stolen credentials
- Scraping publisher sites in violation of ToS
- Sharing subscription-accessed content beyond license terms

## Version Recording

When using non-published versions, record:
- Version type: preprint / accepted-manuscript / tech-report / author-copy
- Key differences from published version (if known)
- Whether figures/tables may differ
- DOI of published version for reference

## When No Full Text Available

Options (in order):
1. Proceed with abstract + metadata only (mark `full_text_available: false`)
2. Ask user to manually obtain and upload
3. Check interlibrary loan options
4. Skip this source and find alternatives
